Building a School Management Portal from Scratch

Japhet Sunday · June 23, 2026 · 10 min read Updated

EduCore started as a capstone project idea: what would a real school admin portal look like if you built it with no backend? Turns out you can get surprisingly far — full CRUD for students, teachers, courses, results, and an attendance tracker — using only three files and localStorage.

Scope of the Project

Before writing a line of code, I listed every feature the portal needed:

  • Student registry with add / edit / delete
  • Teacher profiles and course assignments
  • Results entry and grade calculation
  • Attendance tracker with present / absent / late states
  • Dark mode toggle that persists across sessions
  • Realistic Nigerian seed data so the demo looks credible

Three-File Architecture

EduCore ships as exactly three files: index.html, style.css, and app.js. Bootstrap 5 is loaded from a CDN. This constraint forces you to think carefully about separation of concerns even without a build tool enforcing it.

The Seven localStorage Keys

Each data domain gets its own storage key rather than one giant serialised object. This makes reads faster and avoids overwriting unrelated data on every save: 

// constants.js — one place for all key names
const KEYS = {
  students:   'educore_students',
  teachers:   'educore_teachers',
  courses:    'educore_courses',
  results:    'educore_results',
  attendance: 'educore_attendance',
  darkMode:   'educore_dark_mode',
  lastSaved:  'educore_last_saved',
};

Dynamic Table Rendering

Every table in the portal is rendered by a single renderTable(key, columns) function. Pass it a storage key and a column config array; it reads from localStorage, maps the data to rows, and injects the HTML. Adding a new data domain means adding one key and one column config — not a new render function. 

function renderTable(key, columns) {
  const data = JSON.parse(localStorage.getItem(key)) ?? [];
  const tbody = document.querySelector(`[data-table="${key}"] tbody`);

  tbody.innerHTML = data.map(row => `
    <tr>
      ${columns.map(col => `<td>${row[col.field] ?? '—'}</td>`).join('')}
      <td class="actions">
        <button onclick="editRow('${key}', '${row.id}')">Edit</button>
        <button onclick="deleteRow('${key}', '${row.id}')">Delete</button>
      </td>
    </tr>
  `).join('');
}

Dark Mode Without a Framework

Dark mode is implemented with a single attribute on <body>: data-theme="dark". CSS custom properties do the rest — every colour is defined as a variable, so the entire palette swaps with one attribute change. The preference is read before the first paint to prevent a flash of light mode. 

// Read preference synchronously, before any rendering
const savedTheme = localStorage.getItem('educore_dark_mode');
if (savedTheme === 'dark') {
  document.body.setAttribute('data-theme', 'dark');
}

// Toggle on button click
function toggleDarkMode() {
  const isDark = document.body.getAttribute('data-theme') === 'dark';
  document.body.setAttribute('data-theme', isDark ? 'light' : 'dark');
  localStorage.setItem('educore_dark_mode', isDark ? 'light' : 'dark');
}

Seed Data Strategy

On first load, if localStorage is empty, the app runs a seedDatabase() function that populates all seven keys with realistic-looking Nigerian school data. This makes every demo feel live rather than empty — critical when showing the project to a recruiter.

Lessons Learned

The biggest surprise: how far a disciplined module-like structure takes you in vanilla JS. The second biggest: Bootstrap's utilities handle 80% of layout needs — you only write custom CSS for your brand's 20%.
