Building a School Management Portal from Scratch

Japhet Sunday · June 23, 2026 · 10 min read

EduCore started as a capstone project idea: what would a real school admin portal look like if you built it with no backend? Turns out you can get surprisingly far — full CRUD for students, teachers, courses, results, and an attendance tracker — using only three files and localStorage.

Scope of the Project

Before writing a line of code, I listed every feature the portal needed:

  • Student registry with add / edit / delete
  • Teacher profiles and course assignments
  • Results entry and grade calculation
  • Attendance tracker with present / absent / late states
  • Dark mode toggle that persists across sessions
  • Realistic Nigerian seed data so the demo looks credible

Three-File Architecture

EduCore ships as exactly three files: index.html, style.css, and app.js. Bootstrap 5 is loaded from a CDN. This constraint forces you to think carefully about separation of concerns even without a build tool enforcing it.

The Seven localStorage Keys

Each data domain gets its own key rather than one giant serialised object. This makes reads faster and avoids overwriting unrelated data on every save: 

const KEYS = {
  students:   'educore_students',
  teachers:   'educore_teachers',
  courses:    'educore_courses',
  results:    'educore_results',
  attendance: 'educore_attendance',
  darkMode:   'educore_dark_mode',
  lastSaved:  'educore_last_saved',
};

Dynamic Table Rendering

Every table in the portal is rendered by a single renderTable(key, columns) function. Pass it a storage key and a column config array; it reads from localStorage, maps the data to rows, and injects the HTML. Adding a new data domain means adding a key and a column config — not a new rendering function.

Dark Mode Without a Framework

Dark mode is implemented with a single class on <body>: data-theme="dark". CSS custom properties do the rest — every colour in the design system is defined as a variable, so the entire palette swaps with one attribute change. The preference is written to localStorage on toggle and read on page load before the first paint to prevent a flash of light mode.

Seed Data Strategy

On first load, if localStorage is empty, the app runs a seedDatabase() function that populates all seven keys with realistic-looking Nigerian school data: proper names, subject combinations that match real curricula, and plausible attendance records. This makes every demo feel live rather than empty.

Lessons Learned

The biggest surprise was how far a disciplined module-like structure gets you in vanilla JS. The second biggest was that Bootstrap's utility classes handle 80% of layout needs — you only write custom CSS for the 20% that requires your brand's visual identity.
