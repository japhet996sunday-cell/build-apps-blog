How to Build an Expense Tracker with Vanilla JS

Japhet Sunday · June 16, 2026 · 8 min read Updated

An expense tracker is one of those projects that sounds simple but hides real engineering decisions. By the end of this walkthrough you'll have a working app that stores data between sessions, filters by category, and renders a live chart — all without a single framework.

What We're Building

BudgetBuddy is a single-page expense tracker. The key features are:

  • Add, edit, and delete transactions with a category label
  • Persistent storage via localStorage
  • A doughnut chart powered by Chart.js showing spending by category
  • Filter transactions by month or category
  • A running balance displayed in the header

Architecture: Seven Modules

Rather than dumping everything into one JavaScript file, BudgetBuddy uses a seven-module architecture with native ES Modules (no bundler required):

  • app.js — entry point, wires everything together
  • storage.js — read/write wrapper around localStorage
  • transactions.js — CRUD operations on transaction data
  • calculations.js — totals, balances, category summaries
  • filters.js — filter state and predicate logic
  • ui.js — all DOM mutations live here, nowhere else
  • utils.js — date formatting, currency formatting, etc.

The Storage Module

The golden rule of storage.js: it is the only file that touches localStorage. Every other module asks it for data; none read from window.localStorage directly. 

// storage.js
const STORAGE_KEY = 'budgetbuddy_transactions';

export function loadTransactions() {
  try {
    return JSON.parse(localStorage.getItem(STORAGE_KEY)) ?? [];
  } catch {
    return [];
  }
}

export function saveTransactions(transactions) {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(transactions));
}

The try/catch is not optional — JSON.parse throws if the stored value is corrupt, and browsers can throw a QuotaExceededError on writes when storage is full.

Chart.js Integration

Load Chart.js from a CDN and initialise the doughnut chart once on page load. When the transaction list changes, update the data arrays and call chart.update() — never destroy and recreate the chart instance, as that causes a visible flash.

Setting Up the Canvas

// In your HTML: <canvas id="spending-chart"></canvas>

const ctx = document.getElementById('spending-chart').getContext('2d');
const chart = new Chart(ctx, {
  type: 'doughnut',
  data: { labels: [], datasets: [{ data: [], backgroundColor: [] }] },
  options: { responsive: true, maintainAspectRatio: false }
});

Updating Without Flashing

function updateChart(categoryTotals) {
  chart.data.labels = Object.keys(categoryTotals);
  chart.data.datasets[0].data = Object.values(categoryTotals);
  chart.update(); // redraws smoothly — no destroy/recreate
}

The ES Module Gotcha

If you open index.html directly from the filesystem (file://), the browser will refuse to load ES Modules and throw a CORS error. Always serve the project through a local server — VS Code's Live Server extension or npx serve . both work.
