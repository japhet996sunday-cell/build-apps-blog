Crafting a Responsive Navbar That Actually Works

Japhet Sunday · June 30, 2026 · 7 min read Updated

Navigation is one of those components that every tutorial shows wrong — a hamburger that barely opens, no keyboard support, no scroll-awareness, and active links that never actually update. Let's build one that handles all of it.

The HTML Foundation

Start with a <header> containing a logo link, a toggle button, and a <nav>. The toggle needs aria-expanded to communicate open/closed state to screen readers, and aria-controls pointing to the nav's id. 

<header class="site-header" data-site-header>
  <a href="/" class="site-logo">My Site</a>

  <button
    class="nav-toggle"
    data-nav-toggle
    aria-expanded="false"
    aria-controls="site-nav"
    aria-label="Toggle navigation"
  >
    <span aria-hidden="true"></span>
    <span aria-hidden="true"></span>
    <span aria-hidden="true"></span>
  </button>

  <nav id="site-nav" class="site-nav" aria-label="Main navigation">
    <ul role="list">
      <li><a href="#about">About</a></li>
      <li><a href="#projects">Projects</a></li>
    </ul>
  </nav>
</header>

The Hamburger Toggle

Three empty <span> elements become the hamburger bars. CSS transforms animate them into an × when aria-expanded="true" is set. Tying the animation to the ARIA attribute means the visual state and the accessible state can never get out of sync. 

/* Three bars */
.nav-toggle span {
  display: block;
  width: 22px;
  height: 2px;
  background: currentColor;
  transition: transform 0.25s, opacity 0.25s;
}

/* Open state: animate into × */
.nav-toggle[aria-expanded="true"] span:first-child {
  transform: translateY(8px) rotate(45deg);
}
.nav-toggle[aria-expanded="true"] span:nth-child(2) {
  opacity: 0;
  transform: scaleX(0);
}
.nav-toggle[aria-expanded="true"] span:last-child {
  transform: translateY(-8px) rotate(-45deg);
}

Scroll-Aware Header

Two scroll states make a header feel polished. Scrolled adds a backdrop and shadow after 60px. Hidden slides the header off-screen when scrolling down past the fold, and brings it back on scroll-up. 

let lastScrollY = window.scrollY;

window.addEventListener('scroll', () => {
  const y = window.scrollY;

  // Add glass effect after 60px
  header.classList.toggle('is-scrolled', y > 60);

  // Hide on scroll-down (past the fold), show on scroll-up
  if (y > 200) {
    header.classList.toggle('is-hidden', y > lastScrollY);
  } else {
    header.classList.remove('is-hidden');
  }

  lastScrollY = y;
}, { passive: true }); // passive: true = no scroll jank

Active Link Tracking with IntersectionObserver

Comparing window.scrollY to section offsets on every scroll event is imprecise and expensive. IntersectionObserver is better: observe each section, and when one enters the target zone, mark its nav link active. 

const sections = document.querySelectorAll('section[id]');
const navLinks = document.querySelectorAll('.site-nav a[href^="#"]');

const observer = new IntersectionObserver(entries => {
  entries.forEach(entry => {
    if (!entry.isIntersecting) return;

    // Deactivate all, then activate the matching link
    navLinks.forEach(link => {
      const isActive = link.getAttribute('href') === `#${entry.target.id}`;
      link.classList.toggle('is-active', isActive);
      link.setAttribute('aria-current', isActive ? 'true' : 'false');
    });
  });
}, {
  // Only trigger when the section is in the middle 5% of the viewport
  rootMargin: '-40% 0px -55% 0px',
});

sections.forEach(section => observer.observe(section));

Why Those rootMargin Values?

-40% 0px -55% 0px shrinks the detection zone to a 5% band in the middle of the viewport. A section only becomes "active" when it's clearly dominant on screen — not merely when its top pixel enters the viewport, which would mark the next section active prematurely.

Keyboard & Focus Management

When the mobile menu closes, return focus to the toggle button. If the menu is an overlay covering the full screen, trap focus inside it — the Tab key should cycle through the menu items, not invisible links behind the overlay. 

function closeNav() {
  nav.classList.remove('is-open');
  toggle.setAttribute('aria-expanded', 'false');
  document.body.classList.remove('nav-open');
  toggle.focus(); // Return focus to the button that opened the menu
}

// Close on Escape key
document.addEventListener('keydown', e => {
  if (e.key === 'Escape' && nav.classList.contains('is-open')) {
    closeNav();
  }
});

The Body Scroll Lock

Without it, the page scrolls behind the mobile overlay — disorienting and always missed in desktop testing: 

/* style.css */
body.nav-open { overflow: hidden; }

/* JavaScript — toggle the class on open/close */
document.body.classList.toggle('nav-open', isOpen);
